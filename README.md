# Onboarding System — מעבדת הקרב

מערכת הכנסת ניסוי — תרגום שפת לקוח לשפת פיתוח.

## מבנה הפרויקט

```
onboarding-app/
├── index.html        # HTML ראשי
├── src/
│   ├── styles.css    # כל ה-CSS
│   ├── data.js       # קטלוגים ונתוני דמו
│   └── app.js        # לוגיקה מלאה
└── README.md
```

## הפעלה

פשוט פתח את `index.html` בדפדפן — אין צורך ב-build tools או npm.

## Flow מרכזי

### צד לקוח (טאב: לקוח / מנהל מוצר)
1. **שלב 1** — בחירת קטגוריה (אוויר / קרקע / ...)
2. **שלב 2** — בחירת אפליקציה (מטוס תקיפה / רחפן תצפית / ...)
3. **שלב 3** — צפייה ועריכת יכולות של האפליקציה
4. לחיצה על **"צור קבלה"** → מוצג סיכום מלא
5. לחיצה על **"שלח לפיתוח"** → עובר לטאב פיתוח

### צד פיתוח (טאב: פיתוח)
- Drill-down: אפליקציה → יכולות → תת-יכולות → פרמטרים
- סטטוסים: קיים (ירוק) / התאמה (כתום) / פיתוח (אדום)
- הוספת יכולות ותת-יכולות מקטלוג מוגדר

## אינטראקציות
- **לחיצה ימנית** על כל פריט → תפריט עריכה/מחיקה
- **+ יכולת** → popup עם קטלוג + אפשרות חדשה
- **+ תת-יכולת** → popup עם חיפוש מכל התת-יכולות הקיימות
- **+ פרמטר** → הוספה ידנית

| סטטוס | צבע | משמעות |
|-------|-----|---------|
| קיים | ירוק `#3B6D11` | קיים ומוכן לשימוש |
| התאמה | כתום `#BA7517` | קיים, דורש התאמה קטנה |
| פיתוח | אדום `#A32D2D` | דורש פיתוח חדש |

## הוספת נתונים
כדי להוסיף קטגוריות, אפליקציות, יכולות — ערוך את `src/data.js`:
- `CAP_CATALOG` — רשימת כל היכולות האפשריות
- `ALL_SUBS` — רשימת כל התת-יכולות עם שיוך ליכולת אם
- `PAR_CATALOG` — פרמטרים ברירת מחדל לכל תת-יכולת
- `DB` — נתוני ברירת המחדל הראשוניים
